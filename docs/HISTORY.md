# 프로젝트 변천사

## 주요 변경 기록

### 2026-04-21 — UX 고도화 및 하이엔드 디자인 개편
- **Apple-Style UX**:
    - **솨랴랴한 애니메이션**: 스크롤을 내릴 때 버튼이 아래로 숨겨지고, 살짝 올릴 때 다시 나타나는 "Show on Scroll Up" 패턴 적용.
    - **Quintic Easing**: 모든 트랜지션에 `cubic-bezier(0.16, 1, 0.3, 1)`을 적용하여 애플 특유의 부드럽고 쫀득한 모션 구현.
- **하이엔드 디자인**:
    - **Slate Silver 테마**: 촌스러운 블랙/블루 대신 세련된 `#515154` (슬레이트 그레이)와 투명도가 적용된 영롱한 블루 포인트 사용.
    - **곡률 통일**: 모든 버튼(추가, 조회)의 곡률을 `12px`로 통일하고 시각적 정체성 일원화.
- **기능 개선**:
    - 리스트 하단에 도달 시 플로팅 버튼이 사라지고 정적 버튼과 자연스럽게 교체되는 로직 구현.
    - `ApiResult` 타입 임포트 방식 수정(`import type`)으로 런타임 오류 해결.


---

## 1기 — 초기 프로토타입 (2024~2026-03)
- **특징**: 단일 파일(`App.tsx`)에 모든 UI와 로직이 포함된 형태.
- **기능**: 기본적인 키워드 추가/삭제 및 네이버 검색량 조회 API 연동.
- **상태 관리**: 단순 `useState` 기반, 별도의 컴포넌트 구조 없음.

---

## 2기 — 기능 고도화 및 DnD 도입 (2026-04 초반)
- **DnD 도입**: `@dnd-kit`을 활용한 키워드 순서 변경(Drag & Drop) 기능 추가.
- **일괄 입력**: 단일 입력 외에 쉼표(,)나 줄바꿈을 활용한 키워드 일괄 추가 기능 구현.
- **디자인**: 다크 모드/라이트 모드 지원 및 CSS 변수(`--accent` 등) 기반 테마 시스템 구축.

---

## 3기 — 현재: 아키텍처 정립 (2026-04-21 ~ )
- **구조**: `src/components/` 폴더 기반의 컴포넌트 주도 개발 환경 구축.
- **성능**: 불필요한 리렌더링 방지를 위한 컴포넌트 단위 최적화.
- **환경**: `Vite` + `TypeScript` + `Netlify` 배포 환경 안정화.
