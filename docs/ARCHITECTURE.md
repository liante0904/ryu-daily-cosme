- **Framework**: React 18 (TypeScript)

- **DnD**: @dnd-kit (Core, Sortable)
- **Deployment**: Netlify

- **Method**: POST (키워드 배열 전송), GET (히스토리 조회)

## 아키텍처 결정 기록 (ADR)

### ADR-001: 컴포넌트 기반 리팩토링
- **배경**: 단일 파일(`App.tsx`)의 코드 비대화로 인한 유지보수성 저하.
- **결정**: UI 요소를 `Header`, `Sidebar`, `KeywordItem` 등으로 분리하여 독립적으로 관리.

### ADR-002: 플로팅 액션 버튼(FAB) UX 고도화
- **배경**: 키워드 목록의 길이에 상관없이 즉각적인 조회 접근성이 필요함.
- **결정**: 
    - 버튼을 화면 중앙 하단에 고정(`fixed`).
    - **상시 노출**: 키워드가 없는 초기 상태에서도 버튼 위치를 유지하여 사용자가 '조회' 기능을 즉시 인지하도록 함(단, 키워드 0개 시 비활성화).
    - **콘텐츠 우선순위**: 스크롤 시 사라지는 애니메이션을 통해 콘텐츠 탐색을 방해하지 않음.

### ADR-005: PC 지향형 인터랙션 디자인
- **배경**: 주로 PC에서 사용되는 페이지 특성상 마우스 조작 피드백과 가독성이 중요함.
- **결정**:
    - 액션 아이콘 호버 시 확대 피드백 제공.
    - 수정 모드 진입 시 해당 행(Row)을 물리적으로 확대하고 폰트 크기를 키워 입력 집중도를 높임.

### ADR-003: 테마 시스템 구현
- **배경**: 사용자 환경(OS 설정) 및 수동 전환 기능을 모두 지원해야 함.
- **결정**: `data-theme` 속성과 CSS 변수를 활용하여 무중단 테마 전환 및 `localStorage`를 통한 설정 저장 구현.

### ADR-004: 데이터 입출력 유연화
- **배경**: 대량의 키워드를 입력하거나 기존 리스트를 복사해야 하는 니즈 발생.
- **결정**: "단일 입력"과 "일괄 입력(CSV/줄바꿈)" 모드를 토글 방식으로 제공.

## 향후 로드맵
- [ ] **Custom Hooks 도입**: `App.tsx`에 남은 데이터 처리 로직을 `useKeywords` 등으로 완전 분리.
- [ ] **결과 시각화 고도화**: 단순 JSON 출력에서 테이블 또는 차트 형태로 분석 결과 시각화.
- [ ] **PWA 지원**: 오프라인에서도 키워드 목록을 확인하고 관리할 수 있도록 지원.
